# 卡牌PVE游戏项目

这是一个基于Unity的回合制卡牌PVE游戏项目。

## 功能
- 回合制卡牌战斗系统
- 全局地图系统
- 技能组合系统
- 可随时随地访问
- 版本控制

## 项目生成需求

### 卡牌 PVE 游戏开发需求提示词

请根据以下完整规则，开发一款 回合制卡牌 PVE 游戏 ，使用Unity 实现，代码结构清晰、可直接运行，界面简洁可用，逻辑严格按照下面描述实现：

### 一、整体游戏结构（参考杀戮尖塔）
- 游戏包含 全局地图系统
- 地图为 分支路线节点结构 ，玩家可选择前进方向
- 节点类型包括：
  - 怪物战斗节点
  - 商店节点（可购买 / 删除卡牌）
  - 随机事件节点（对话选择，获得奖励 / 惩罚）
- 玩家从起点出发，逐步解锁节点，通关最终 BOSS 获胜

### 二、核心战斗系统规则
- 采用 回合制战斗 ：玩家回合 → 怪物 AI 回合 循环
- 拥有 法力费用系统 ：每回合固定获得若干费用，出牌消耗费用
- 战场中间有一个 横槽（卡牌序列槽） ，默认长度为 10
- 卡牌类型只有三种：
  - 攻击卡
  - 防御卡
  - 治疗卡
- 出牌规则：
  - 每张卡牌消耗 1 点费用
  - 打出后 放入横槽任意位置
  - 放入后，该位置右侧原有卡牌 全部顺次右移一格
  - 若右移后超过横槽长度上限（10）， 最右侧卡牌被永久删除

### 三、技能系统（核心特色机制）
- 每个技能由 2～4 个效果类型排列  组成，例如：
  - 攻击 → 防御
  - 防御 → 治疗 → 攻击
  - 攻击 → 攻击 → 防御 → 治疗
- 触发条件：
  - 横槽中 连续存在与技能完全相同的子序列 时可触发
- 技能有两种使用模式：
  - 施放模式
    - 触发技能效果
    - 消耗对应序列（可设定删除或保留）
    - 回复 1 点费用
  - 蓄力模式
    - 不立即触发效果
    - 将本次技能 存储起来
    - 回复 1 点费用
- 蓄力存储规则：
  - 存储的技能可在 当前回合任意时刻无消耗施放一次
  - 本次施放 不再回复费用

### 四、战斗基础规则
- 玩家与怪物各有生命值
- 攻击卡：对敌方造成伤害
- 防御卡：为玩家增加护盾
- 治疗卡：恢复玩家生命值
- 怪物 AI 按固定逻辑行动：攻击 / 技能 / 防御
- 一方生命值≤0 则战斗结束

### 五、开发要求
- 输出完整可运行代码
- 界面包含：
  - 玩家血条、护盾、费用
  - 怪物血条
  - 手牌区
  - 横槽展示区（长度 10）
  - 技能列表与蓄力技能区
  - 地图路线界面
- 逻辑严格遵循上述规则，不随意简化核心机制
- 代码注释清晰，便于后续扩展
- 先实现最小可玩版本，再逐步扩展内容
